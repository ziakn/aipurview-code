# Audit: reporting/generating-reports
**Article path:** shared/user-guide-content/content/reporting/generating-reports.ts
**Audited:** 2026-04-29
**Auditor:** Explore subagent (v2)
**Verdict:** ⚠️ minor issues (2)

## Summary
The article is largely accurate in its description of report capabilities, table columns, and features. Two UI claims are incorrect: the article refers to non-existent buttons "Generate use case report" and "Generate organization report," when the actual UI is a single "Generate report" button with a dropdown menu. The table column labels are verified as accurate.

## Findings

### Finding 1 — "Generate use case report" and "Generate organization report" buttons don't exist
- **Type:** UI
- **Status:** ❌ wrong
- **Doc says:** "Click 'Generate use case report'" (block 123) and "Click 'Generate organization report'" (block 157)
- **Reality:** The UI has a single "Generate report" button that opens a dropdown menu with options labeled "Use case report" and "Organization report"
- **Evidence:** `Clients/src/presentation/pages/Reporting/GenerateReport/index.tsx:88` shows button labeled "Generate report"; lines 24-37 define menu options as "Use case report" and "Organization report"
- **Suggested fix:** Change block 123 from "Click 'Generate use case report'" to "Click 'Generate report'" and select the use case report type from the dropdown. Same for block 157 with organization report.
- **Confidence:** high

### Finding 2 — Step ordering mismatch for use case report generation
- **Type:** Behavior
- **Status:** ⚠️ partial
- **Doc says:** "Click 'Generate use case report'" followed by "Select the use case from the dropdown" (blocks 123-124)
- **Reality:** Users click "Generate report" button, select a type from the popover menu, then the modal opens where they select the use case
- **Evidence:** `Clients/src/presentation/pages/Reporting/GenerateReport/index.tsx:54-68, 114-140` shows popover menu with report type selection precedes project selection in the modal
- **Suggested fix:** Reorder the steps to reflect: (1) Click "Generate report", (2) Select report type from dropdown, (3) Select use case in modal
- **Confidence:** high
